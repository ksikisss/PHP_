# PHP Packages Demo

Цей проект демонструє роботу трьох популярних PHP пакетів, які дуже часто використовуються у веб-розробці. Кожен пакет має свою спеціалізацію і допомагає вирішувати конкретні завдання в програмуванні.

## Що таке пакети та навіщо вони потрібні?

**Пакети** - це готові бібліотеки коду, які інші розробники написали та поділилися з community. Вони допомагають не писати все з нуля, а використовувати перевірені рішення. Це як використовувати готові інструменти замість того, щоб робити їх самостійно.

У цьому проекті ми розглянемо три пакети:
- **Symfony VarDumper** - для відлагодження та виведення даних
- **Monolog** - для логування (запису подій)
- **Carbon** - для роботи з датами та часом

## Файли демонстрації

### 1. var_dumper_demo.php - Symfony VarDumper

**Що таке Symfony VarDumper?**
Це інструмент для розробників, який допомагає "побачити" що знаходиться всередині змінних у PHP. Звичайно, коли ви пишете код і щось не працює, вам потрібно подивитися, які дані містяться у ваших змінних. VarDumper робить це красиво та зрозуміло.

**Для чого використовується?**
- Під час розробки, коли потрібно перевірити, які дані передаються у функції
- Для відлагодження помилок - щоб зрозуміти, що саме знаходиться у змінній
- Для вивчення структури даних, які приходять з бази даних або API

**Що робить наш демо-файл:**
1. **Показує масив з різними даними** - ім'я, вік, email, хобі людини
2. **Демонструє різні типи даних** - рядки, числа, булеві значення, null, масиви, об'єкти
3. **Показує виведення об'єктів** - створює об'єкт користувача з властивостями
4. **Використовує функцію dd()** - це "dump and die", тобто вивести дані і зупинити виконання скрипта

**Приклад з життя:**
Уявіть, що ви пишете інтернет-магазин. Користувач додає товар у кошик. Ви хочете перевірити, чи правильно зберігаються дані про товар. VarDumper допоможе вам красиво вивести всю інформацію про товар: назву, ціну, кількість, тощо.

**Запуск:**
```bash
php var_dumper_demo.php
```

### 2. monolog_demo.php - Monolog

**Що таке Monolog?**
Це система логування для PHP. **Логування** - це запис подій, які відбуваються у вашій програмі. Це як щоденник вашого додатку, де записується все важливе: помилки, успішні операції, попередження тощо.

**Для чого використовується логування?**
- **Відстеження помилок** - коли щось ламається, логи допоможуть знайти причину
- **Моніторинг роботи** - можна бачити, як працює додаток у продакшені
- **Аудит** - запис важливих дій (вхід користувачів, зміни даних)
- **Відлагодження** - можна відстежити послідовність виконання коду

**Рівні логування (від найнижчого до найвищого):**
- **DEBUG** - детальна інформація для розробки
- **INFO** - загальна інформація про роботу
- **NOTICE** - важливі події, які не є помилками
- **WARNING** - попередження про потенційні проблеми
- **ERROR** - помилки, які не зупиняють роботу
- **CRITICAL** - критичні помилки
- **ALERT** - потрібно негайно реагувати
- **EMERGENCY** - система не працює

**Що робить наш демо-файл:**
1. **Базове логування** - створює логер і записує повідомлення різних типів
2. **Ротація лог-файлів** - коли файл стає занадто великим, створюється новий (лог1, лог2, тощо)
3. **Процесори** - додають додаткову інформацію до логів:
   - Скільки пам'яті використовує скрипт
   - Інформація про веб-запит (URL, IP-адреса, браузер)
4. **Різні рівні логування** - демонструє всі 8 рівнів

**Приклад з життя:**
Уявіть інтернет-магазин. Monolog може записувати:
- "Користувач user123 увійшов у систему" (INFO)
- "Не вдалося підключитися до бази даних" (ERROR)
- "Замовлення #12345 успішно оформлено" (INFO)
- "Спроба злому: неправильний пароль 5 разів" (WARNING)

Це допомагає адміністраторам стежити за роботою сайту і швидко реагувати на проблеми.

**Запуск:**
```bash
php monolog_demo.php
```

Логи будуть збережені в папці `logs/`.

### 3. carbon_demo.php - Carbon

**Що таке Carbon?**
Це потужна бібліотека для роботи з датами та часом у PHP. PHP має вбудовані функції для дат, але вони незручні. Carbon робить роботу з датами інтуїтивною та простою.

**Для чого використовується?**
- **Розрахунки дат** - додати дні/місяці/роки до дати
- **Форматування** - вивести дату у будь-якому потрібному форматі
- **Порівняння** - перевірити, яка дата більша/менша
- **Часові пояси** - конвертувати час між різними містами
- **Людські інтервали** - "3 дні тому", "через 2 години"

**Основні можливості:**
- Створювати дати з рядків, чисел, або поточного часу
- Додавати/віднімати час (дні, години, хвилини)
- Форматувати у різних стилях (ISO, український, тощо)
- Працювати з різними часовими поясами
- Обчислювати різницю між датами

**Що робить наш демо-файл:**
1. **Створення дат** - поточний час, конкретна дата, з рядка
2. **Форматування** - ISO формат, людський формат, українською мовою
3. **Математика з датами** - додати тиждень, місяць, відняти дні
4. **Порівняння** - яка дата більша, різниця в днях
5. **Інтервали часу** - створити період "3 дні 5 годин 30 хвилин"
6. **Часові пояси** - конвертувати між UTC, Києвом, Лондоном
7. **Корисні методи** - день тижня, місяць, чи рік високосний

**Приклад з життя:**
У соціальній мережі Carbon використовується для:
- "Опубліковано 2 години тому"
- "Народився 15 березня 1990 року"
- "Подія відбудеться через 3 дні"
- "Робочий час: 9:00 - 18:00"

Без Carbon такі речі доводилося б рахувати складними математичними формулами з unix timestamps.

**Запуск:**
```bash
php carbon_demo.php
```

## Чому саме ці пакети?

**Symfony VarDumper** - стандарт для відлагодження у Symfony фреймворку, але працює і окремо
**Monolog** - найпопулярніша система логування для PHP, використовується у Laravel, Symfony
**Carbon** - де-факто стандарт для роботи з датами, має величезну community

## Порівняння з вбудованими засобами PHP

| Завдання | PHP (вбудований) | Наш пакет |
|----------|------------------|-----------|
| Виведення змінних | `var_dump()` (некрасиво) | `dump()` (красиво) |
| Логування | Немає стандартного | Monolog (професійне) |
| Робота з датами | `date()`, `strtotime()` (складно) | Carbon (просто) |
